    Veiled Cars

    Some car files were encrypted - waiting to be unveiled in real-life before being playable ingame. These files also had a custom volume flag that is tied to the MBufferDecryptManager adhoc object.

    In order for the car files to be decrypted, the Grim server returned decryption keys on release day. These keys were stored in the save, and MBufferDecryptManager.addKey() would be called, making these files available.

    If an unlocked game save wasn't backed up from back in the day, these files would've been permanently inaccessible.
